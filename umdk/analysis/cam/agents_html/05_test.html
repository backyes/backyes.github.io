def SKIP_ENV_RANKSIZE_UNEQUAL(expect_ranksize, reason='skip if ranksize is not equal to '):
    skip = False
    if expect_ranksize != tool.get_world_size():
        skip = True
        reason += str(expect_ranksize)

这是一个运行时条件跳过装饰器：当实际 WORLD_SIZE 不等于期望值时自动 skip。例如 @SKIP_ENV_RANKSIZE_UNEQUAL(4) 表示仅在 4 卡环境执行。

ref_weights = np.where(topk_idx_np == -1, np.float32(0), topk_weights_np).sum(axis=1).reshape(-1, 1)

# 相似度计算（余弦相似度变体）
def calc_diff(x, y):
    x, y = x.astype(np.float64) + 1, y.astype(np.float64) + 1
    denominator = (x * x + y * y).sum()
    sim = 2 * (x * y).sum() / denominator
    return 1 - sim

assert(diff < 5e-5)

3.4 FP8 反量化实现

def per_token_cast_back(x_fp8: torch.Tensor, x_scales: torch.Tensor):
    if x_scales.dtype == torch.int:
        x_scales = x_scales.view(dtype=torch.int8).to(torch.int) << 23
        x_scales = x_scales.view(dtype=torch.float)
    x_fp32 = x_fp8.to(torch.float32).view(x_fp8.size(0), -1, 128)
    x_scales = x_scales.view(x_fp8.size(0), -1, 1)
    return (x_fp32 * x_scales).view(x_fp8.shape).to(torch.bfloat16)

这是 128-element per-tile FP8 反量化：将 int8 的 scale 通过位操作还原为 float，然后按 128 元素粒度做 per-token 缩放。

4.2 NPUBridge — Tensor 到 StorageImpl 的桥接

class NPUBridge {
public:
    static NPUStorageImpl *GetNpuStorageImpl(const at::Tensor &tensor);
    static NPUStorageImpl *GetNpuStorageImpl(c10::StorageImpl *storageImpl);
    static NPUStorageImpl *GetNpuStorageImpl(c10::Storage &&storage);
    static NPUStorageDesc &GetNpuStorageImplDesc(const at::Tensor &tensor);
};

实现方式：通过 static_cast 将 PyTorch 的 c10::StorageImpl 强制转换为 CAM 自定义的 NPUStorageImpl。这是侵入式扩展——CAM 在创建 tensor 时用 NPUStorageImpl 替换默认 StorageImpl。

4.3 NPUStorageDesc — NPU 存储元数据

struct NPUStorageDesc {
    c10::SmallVector<int64_t, 5> base_sizes_;
    c10::SmallVector<int64_t, 5> base_strides_;
    c10::SmallVector<int64_t, 5> storage_sizes_;
    int64_t base_offset_ = 0;
    use_byte_size_t base_dtype_ = {};
    aclFormat origin_format_ = ACL_FORMAT_UNDEFINED;
    aclFormat npu_format_ = ACL_FORMAT_ND;
    caffe2::TypeMeta data_type_ = caffe2::TypeMeta::Make<uint8_t>();
};

关键字段： - base_sizes_ / base_strides_：原始 tensor 的 shape/stride - storage_sizes_：实际存储的 storage 维度（考虑 NZ 格式转换） - origin_format_ / npu_format_：ACL 格式（ND、FRACTAL_NZ 等） - data_type_：用于从 StorageImpl 重建 GE tensor

struct NPUStorageImpl : public c10::StorageImpl {

4.5.3 OpApi 动态加载

inline void *GetOpApiFuncAddr(const char *apiName)
{
    static auto custOpApiHandler = GetOpApiLibHandler(GetCustOpApiLibName());  // libcust_opapi.so
    if (custOpApiHandler != nullptr) {
        auto funcAddr = GetOpApiFuncAddrInLib(custOpApiHandler, GetCustOpApiLibName(), apiName);
        if (funcAddr != nullptr) return funcAddr;
    }
    static auto opApiHandler = GetOpApiLibHandler(GetOpApiLibName());  // libopapi.so
    return GetOpApiFuncAddrInLib(opApiHandler, GetOpApiLibName(), apiName);
}

双库回退：优先查找 libcust_opapi.so（自定义算子 API），找不到再回退到 libopapi.so（标准 ACLNN API）。

4.5.4 EXEC_NPU_CMD 宏 — 算子调用统一入口

#define EXEC_NPU_CMD(aclnn_api, ...)

这是整个 helper 的核心宏，封装了完整的 ACLNN 算子调用流程：

  1. 获取函数地址：GetOpApiFuncAddr 动态加载 aclnn_api 和 aclnn_apiGetWorkspaceSize
  2. 初始化大内存：InitHugeMemThreadLocal
  3. 类型转换：ConvertTypes(__VA_ARGS__, workspace_size_addr, executor_addr) 将所有参数转为 ACL 类型
  4. 获取 Workspace：调用 GetWorkspaceSize 获取所需 workspace 大小
  5. 分配 Workspace：通过 at::empty 在 NPU 上分配
  6. 执行算子：调用 OpApiFunc，传入 workspace 和 acl_stream
  7. 释放资源：ReleaseConvertTypes + UnInitHugeMemThreadLocal + ReleaseHugeMem

    } else {
        // 根据维度数推断格式：3D→NCL, 4D→NCHW, 5D→NCDHW
    }

    aclCreateTensor(sizes, dimNum, acl_data_type, strides, offset, format, storageDims, ...);

8.2.2 容差选择的合理性

  • rtol=0.02, atol=2.0：对于 int8 量化 MoE（hidden_size=7168），2.0 的绝对误差容忍度对应约 0.03% 的相对误差（假设值域 [-5,5]），是合理的量化精度验证
  • 5e-5 余弦相似度：对于 dispatch/combine（纯通信 + 加权），要求几乎无损

九、关键发现与建议

9.1 架构级发现

  1. CAM 采用"Python 测试 + C++ 算子"的混合模式：测试逻辑完全在 Python 层，通过 pybind 调用 C++ 算子。这种模式开发效率高，但无法覆盖 C++ 内部逻辑。

  2. NPUBridge 是侵入式设计：通过 static_cast 将 c10::StorageImpl 转为 NPUStorageImpl，依赖 CAM 自己创建 tensor。如果 PyTorch 内部实现变化（如 StorageImpl 布局调整），可能导致隐蔽 bug。

  3. pytorch_npu_helper.hpp 是完整的 ACL 抽象层：EXEC_NPU_CMD 宏封装了完整的算子调用生命周期（workspace 管理、大内存初始化、类型转换、错误处理），设计成熟。

  4. FP8/FP4 支持有版本兼容设计：通过 CAM_ACL_HAS_FP8_FP4 宏实现新旧 CANN 的兼容编译。

9.2 测试策略建议

  1. 短期： - 启用 CASE_8RANK 测试 - 添加 with_share=True、with_smooth=True、with_mc2_mask=True 的参数化组合 - 实现 run_with_cov.sh 覆盖率收集

  2. 中期： - 为 C++ 核心逻辑（NPUBridge、StorageImpl、类型转换）添加 gtest - 添加 Fuzz 测试覆盖随机 shape/config - 接入 CI Pipeline（Jenkinsfile）

  3. 长期： - 构建 CAM 集成测试套件（参考 URMA 框架） - 添加端到端训练验证（小模型 + 真实数据） - 建立性能回归测试（吞吐量/延迟基线）
