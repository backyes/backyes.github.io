2.3 架构创新

CDNA 5 关键架构特性
  • Wave32 执行：首次从 RDNA (游戏架构) 引入 CDNA，标志 AMD UDNA 统一架构的技术基础。Wave32 对 AI 推理特别有益：更细粒度的 wavefront 意味着在推理负载 (通常 batch size 较小、延迟敏感) 下能更好地利用计算资源。
  • Tensor Data Mover (TDM) 增强：Direct DRAM→WGP LDS 数据搬运，无需中间暂存，减少数据搬运开销
  • 多播内存读取 (Multicast Memory Reads)：单次读取可放大到多个 WGP (类似 NVIDIA Hopper 架构)，最高 4X 带宽放大
  • 更大的缓存：192 MB L2 (3X vs MI355X) + 96 MB LDS (2X vs CDNA 4) + 64 KB L1/WGP (2X)
  • CDNA 4 Infinity Cache 放弃：用更小但更高带宽的 L2 替代
  • UALoE scale-up fabric: 72-GPU 共享内存 pod 设计，而非传统 8-GPU 节点
  • 块缩放低精度数据类型: OCP MXFP4 / MXFP6 / MXFP8 硬件支持
  • 主机接口: 16-lane AMD Infinity Fabric, 256 GB/s 双向
  • SR-IOV: 4 内存分区
  • 视频解码: 4 引擎 HEVC/H.265, AVC/H.264, VP9, AV1; JPEG/MJPEG Codec: 40 核
  • Work-group clustering: 工作组集群
  • Split barriers: 分离屏障
  • Data prefetching: 数据预取
  • New command processor: 新命令处理器，减少调度延迟
🔬 架构洞察 — UDNA 统一架构前兆：AMD 已公开声明计划将 CDNA 和 RDNA 统一为 UDNA (Unified DNA) 架构。CDNA 5 的 Wave32 执行、块缩放低精度数据类型、芯粒封装和 2nm 工艺，都是未来统一架构将继承的技术基础。Wave32 进入 CDNA 意味着 AMD 正在消除游戏和计算架构之间的技术壁垒。

5.2 机架物理架构

Helios 机架物理分解
  • 计算托盘: 18 个 (2 groups × 9 trays, 每托盘 1 OU, 4 GPU + 1x 96-core EPYC 9006, 77kg)

  • 网络: Pensando Vulcano 800 AI NIC (每 GPU 3x 800G NIC for scale-out)
  • 互联: UALoE fabric (scale-up) + UALink (scale-out)
  • Form Factor: OCP Open Rack Wide (1.2m wide, 1.3m deep)

6.2 UALoE (UALink over Ethernet)

UALoE 关键特性
  • 桥接机架级低延迟域到更广泛的 scale-out 基础设施
  • 基于标准以太网 — 使用成熟以太网生态系统，非专有互联
  • AMD 贡献于 ESUN (开放行业工作流，推进以太网用于 scale-up AI)
  • Scale-up (Peak) 双向带宽: 3.6 TB/s (每 GPU)
  • 用于 Helios 72-GPU 共享内存 pod 内部互联
  • 单跳拓扑，最小化延迟
  • 每 GPU 通过 18 个独立 UALoE 站点 分布在不同交换托盘上
  • UALoE 接口: 36 × 400Gbit/s = 3.6 TB/s 峰值双向

七、机架网络软件栈 (AFM/AFOS/vPods)

来源: AMD Helios 弹性网络博客

7.1 AMD Fabric Manager (AFM)

  • Kubernetes-based controller cluster on switch hardware
  • 部署自动化、配置、遥测、可观察性
  • Fabric 范围管理 — 非单独管理网络组件
  • 让操作员对整个 scale-up 域有可见性

7.2 AMD Fabric OS (AFOS)

  • SONiC NOS (Network Operating System)
  • 在 scale-up 交换层直接运行
  • 实时控制和可见性
  • 检测并响应 fabric 内硬件事件
  • 与 AFM 配合，将 scale-up 网络从硬件互联转变为可管理、可观察的运营平台

7.3 vPods (Virtual Pods)

  • 软件定义 GPU 分区
  • 将 72-GPU 域分区为更小的 GPU 环境
  • 训练/测试/推理并发工作负载
  • 工作负载隔离和故障遏制
  • 提高基础设施利用率

7.4 硬件弹性

  • 每 GPU 18 个独立 UALoE 站点分布在不同交换托盘上
  • 当一条路径受硬件事件影响时，系统使用备用连接维持统一 GPU 域
  • 自动切换，无需操作员干预，无需终止工作负载
