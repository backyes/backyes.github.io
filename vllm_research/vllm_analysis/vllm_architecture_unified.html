1. Software-Architecture First Principles

PRINCIPLE #0 — The single axiom: "A module boundary should be drawn where the rate of change changes." If A's code changes every day and B's every year, splitting them means "A's daily changes never touch B." If A and B always change together, they belong in the same module.

1.4 First-principles meta-judgement — forced vs chosen

Among all vLLM's layers, a few are forced by physics: paged-KV, Attention-as-separable-backend, Communication-via-Connector, Platform-as-interface — these are stable across HW generations. Others are engineering choices a v2 may freely redesign: the Python front-end, the shm-RPC executor, capability-dispatched sampling, the patch layer. Every heavily-patched subsystem is a gap in vLLM's abstraction surface.

2. Architecture Layers

§2Architecture Layerssource: first_principles_software_arch §2-9 + agent1-3

2.1 EngineCore — the time-line owner

v1/engine/core.py:101 EngineCore. The heartbeat (core.py:576):

def step(self):
    scheduler_output = self.scheduler.schedule()          # 1. decide
    future = self.executor.execute_model(scheduler_output, non_block=True)  # 2. async dispatch
    model_output = future.result()                        # 3. wait
    engine_core_outputs = self.scheduler.update_from_output(scheduler_output, model_output)

EngineCore also owns: add/abort, pause/resume, sleep/wake, LoRA hot-plug, cache reset (reset_prefix_cache/reset_encoder_cache), DP coordination (step_with_batch_queue), spec-draft collection (post_step → take_draft_token_ids). It holds NO device/model/tensor — runs CPU-only, the ultimate control/data-plane split.

2.2 Scheduler — the pure decision function

v1/core/sched/scheduler.py:418. The design axiom (scheduler.py:420):

# There's no "decoding phase" nor "prefill phase" in the scheduler.
# Each request just has num_computed_tokens and num_tokens_with_spec …
# At each step, the scheduler tries to assign tokens to the requests so
# that each request's num_computed_tokens can catch up its
# num_tokens_with_spec. This is general enough to cover chunked prefills,
# prefix caching, speculative decoding, and "jump decoding."

Unified token-budget loop over RUNNING then WAITING, chunked via long_prefill_token_threshold, preemption when allocate_slots returns None. Holds no device/model/comm.

2.3 Worker — the device owner

v1/worker/worker_base.py+gpu_worker.py. One Worker per GPU/NPU. Responsibilities: init_device, load_model/reload_weights, determine_available_memory, kv_caches tensors, sleep/wake, LoRA add/remove/pin, execute_model, sample_tokens, health check. Does NOT hold Scheduler, makes no scheduling decisions, holds no request state — consumes SchedulerOutput statelessly. Worker holds ModelRunner but the two are separate: same device can run different models; same model can run on different devices (Ascend NPUWorker(NPUModelRunner) subclasses but keeps the interface).

2.4 ModelRunner — the forward executor

#stageinput → outputmeaning
1Request bookkeepingSchedulerOutput → req_statesdecisions → local state
2prepare_inputsreq_states → InputBatch{input_ids, positions, q_start_loc, seq_lens, logits_indices}request state → model input tensors
3prepare_attnreq_states+pool → block_tables[groups,reqs,blocks] + slot_mappings[groups,tokens]block ids → page-table walk
4model_state.prepare_attncommon_attn_metadata → attn_metadata[per-backend]common → HW-private metadata

2.5 AttentionBackend — the only HW-sensitive op

Attention is the ONLY op whose memory-access pattern depends on sequence length + KV layout (GEMM/Norm/MoE/RoPE are all regular (N,d)→(N,d)). The triple: register_backend + Builder.build(common) + Impl.forward + KVCacheSpec family (FullAttentionSpec / MLASpec / RSWASpec / RaggedSpec). ModelRunner only sees KVCacheSpec, not concrete model — this decouples "model structure change" from "KV management".

2.6 Platform — device as interface

platforms/. 6 mandatory methods. Holds device capability/memory/stream/attn-backend-selection/graph-pool. Ascend NPUPlatform overrides only identity+backend-map+memory+compile hooks; its get_device_total_memory raises NotImplementedError (torch_npu global-init-only) — a canonical "interface promise vs device capability" mismatch.

2.7 Compilation — "record once, replay many"

compilation/+v1/worker/gpu/cudagraph_utils.py. Platform injects ACLGraphWrapper via get_static_graph_wrapper_cls(). See §6 for the 3-mode trade-off and ACL-specific constraints.

3. Execution Hot Path (Scheduler → ModelRunner → Sample)

§3Hot Pathsource: arch_spine §4 + agent1+agent2 + dsa_analysis

3.1 The page-table walk (slot_mapping kernel)

v1/worker/gpu/block_table.py:283 (_compute_slot_mappings_kernel)
block_indices  = positions // (block_size * CP_SIZE)
block_offsets  = positions %  (block_size * CP_SIZE)
block_numbers  = tl.load(block_table_ptr + req_state_idx*stride + block_indices)
slot_ids = block_numbers * block_size + block_offsets   <- physical address
This line is isomorphic to OS phys = ppn * PAGE_SIZE + page_offset: block_table=page table, block_id=PFN, slot_id=linear address. PAD_SLOT_ID = "invalid PTE". vLLM doing virtual-memory management on GPU HBM, formalised.

3.2 The 3-line heartbeat

def step(self):
    scheduler_output = self.scheduler.schedule()
    future = self.executor.execute_model(scheduler_output, non_block=True)
    engine_core_outputs = self.scheduler.update_from_output(scheduler_output, future.result())

3.3 SchedulerOutput — the Scheduler↔Worker contract

v1/core/sched/output.py: scheduled_new/resumed/running/preempted reqs, req_to_new_blocks, num_scheduled_tokens, scheduled_spec_decode_tokens, scheduled_encoder_inputs, finished_req_ids. Carries ONLY decisions (block ids + token counts) — no tensors — so it crosses process/node boundaries (PD disagg relies on this). 4 explicit categories (new/resumed/running/preempted) let Worker use simple if-else. Block allocation done in Scheduler side — Worker only consumes block ids.

3.4 ForwardContext — the model↔framework firewall

forward_context.py:132: attn_metadata[per-layer], slot_mapping[per-layer], dp_metadata, cudagraph_runtime_mode, batch_descriptor, is_padding, skip_compiled, +all_moe_layers/moe_layer_index (torch.compile string workaround). Model code (200+ files) reads only ForwardContext — changing scheduling/graph-mode/parallelism never touches model files.

3.5 allocate_slots → None — control-flow unification

while True:
    new_blocks = kv_cache_manager.allocate_slots(request, num_new_tokens)
    if new_blocks is not None: break
    preempted = self.running.pop()   # reclaim budget
    self._preempt_request(preempted)

Returning None (not raising) keeps preempt logic in the same while-loop as allocate logic.

4. KV-Cache Management — why 4 layers, not 1

§4KV 4-layer stacksource: arch_spine §2 + agent2 + dsa_analysis + community_deep_dive §4
LayerFileOwnsRate of change
KVCacheManagerv1/core/kv_cache_manager.py:114request-lifecycle ↔ block allocation (allocate_slots/free/get_computed_blocks)daily (policy)
KVCacheCoordinatorv1/core/kv_cache_coordinator.py:60multi-group (MLA+GQA+SWA): logical block id → per-group physical block idsweekly (model structure)

4.1 BlockPool — pure metadata, content-hash

v1/core/block_pool.py:162: blocks = [KVCacheBlock(idx) for idx in range(num_gpu_blocks)], free_block_queue, cached_block_hash_to_block = BlockHashToBlockMap(). KVCacheBlock (kv_cache_utils.py:117) is pure metadata — no tensor. The actual bytes live in Worker-side kv_caches: list[torch.Tensor] (model_runner.py:490). This two-layer split lets prefix-cache/swap/sliding-window/compressed-MLA all renumber the same block_ids without touching the pool.

4.2 Coordinator subclasses

UnitaryKVCacheCoordinator (single group), HybridKVCacheCoordinator (MLA+GQA mixed, e.g. DeepSeek-V3), KVCacheCoordinatorNoPrefixCache. MLA's storage_block_size ≠ block_size mapping is encapsulated here — Manager only calls Coordinator's interface.

4.3 Why 4 layers — the MLA forcing function

v0 had a single BlockSpaceManager. MLA's storage_block_size ≠ block_size added a logical→physical mapping that would have scattered across allocate/free/prefix-cache. After the split, this mapping is encapsulated in Coordinator. "Multi-group mapping" changes are isolated to one layer.

Splitting manager↔coordinator: new KV layout (MLA) changes Coordinator only. Splitting coordinator↔blockpool: prefix-cache/eviction changes BlockPool only. Splitting manager↔connector: PD disagg/offload changes Connector only.

5.1 Context Parallel (PCP/DCP)

VariantWhenComm domainGoal
PCP (Prefill-CP)prefill phaseindependentsplit sequence across devices; reduce TTFT

cp_size = pcp_size × dcp_size; token-interleave storage via cp_kv_cache_interleave_size (default 1); block_size % cp_kv_cache_interleave_size == 0 required.

5.2 EPLB — Ascend's original MoE contribution

Expert-migration stack: EplbUpdator → policy (FlashLB Hungarian + numba tree-search, FlashTree, SwiftBalance) → D2DExpertWeightLoader → VllmEplbAdaptor. The only net-new subsystem from Ascend to upstream vLLM.

5.3 PD disagg — cutting the Worker boundary, no new module

P node owns Scheduler+Worker(P)+KVConnector(P→D); D node owns Scheduler+Worker(D)+KVConnector(D←P). Scheduler only sees KVCacheBlocks (ids), not which node holds the block — "request lifecycle ⟷ KV physical location" decoupling, the direct payoff of KVCacheManager being independent of Scheduler.

6.2 ACL Graph — why Ascend cannot just port CUDA Graph

Source: official docs/source/developer_guide/Design_Documents/ACL_Graph.md

Diff A — full replay needs backend-side update_graph_params() hook (NPU-only). Capture phase records per-graph task handle/event/workspace; replay frontend calls update_full_graph_params() → backend runs parameter refresh on update stream via torch.npu.graph_task_update_begin/end(); ExternalEvent enforces ordering. Key fact: an attn backend WITHOUT this hook CANNOT do full-graph replay. Implemented in vllm_ascend/attention/attention_v1.py, mla_v1.py, context_parallel/attention_cp.py, mla_cp.py.

Diff B — ACL capture can still be rejected by runtime resources. Capture sizes are measured, not estimated from CUDA. Tune cudagraph_capture_sizes / max_cudagraph_capture_size.

6.3 Mode normalization (stricter than upstream)

encoder-decoder forced PIECEWISE; use_inductor disabled on graph path; ASCEND_LAUNCH_BLOCKING=1 rejected when graph enabled; Xlite can disable full mode / fall back to FULL_DECODE_ONLY.

6.4 Fusion & Quant

70+ fusion passes in AscendCompiler/GraphFusionPassManager (AllReduce+RMSNorm, QK-Norm+Rope, AllGather-chunk-noop, NoOp, norm-quant). 10+ quant schemes (W4A4/W4A8/W8A8/PDmix/MXFP/FlatQuant). Rule of thumb: MoE→W4A8, dense→W8A8, extreme(EP off)→W4A4. W4A4∧EPLB incompatible = CANN kernel-coverage limit.

7.2 Three Ascend original innovations

  1. Schedule-level (roofline NPU): RecomputeScheduler (recompute_scheduler.py:98) replaces preemption's offload-to-CPU with drop-and-recompute. Also SRF + profiling-chunk.
  2. EPLB — the net-new vLLB contribution: FlashLB+FlashTree+SwiftBalance, 3-stage updator→policy→D2D migration.
  3. ACL Graph (§6.2) with update_graph_params hook requirement; GraphFusionPassManager 70 passes.

7.3 KV Pool (Ascend Store) — the memory tier

3 backends: Mooncake (Distributed Store + SSD offload), Memcache (MemFabric), Yuanrong (openEuler etcd+HugeTLB). Plus NVMe SSD offload. Mirror of OS cache→DRAM→swapfile. Tuning: kv_load_failure_policy=recompute (hybrid-attn like DSV4 not yet supported); lease TTL ≥ ASCEND_CONNECT_TIMEOUT+TRANSFORM; fabric mem A3 1GB-aligned; PYTHONHASHSEED consistent across nodes; warm-up 2-3× device-count of 8K-input/1-output requests.

7.4 SRF + CPU-binding + Netloader

SRF: anti head-of-line on P-node; threshold=P70-P85 prompt-len, long_max_wait=long-req P90 under strict-short-first. CPU-binding: default-on ARM since v0.18 (uses npu-smi topo to pick NUMA). Netloader: HCCL send/recv weight transfer, register_model_loader API.

7.5 Patch layer = diagnostic of "missing extension points"

Canonical examples: patch_fused_moe (no FusedMoE-factory-swap iface), patch_cudagraph (CudagraphDispatcher assumes CUDA-graph-only), patch_kv_cache_coordinator (internal state not exposed), patch_mla_prefill_backend (no MLA prefill backend-switch iface). Every heavily-patched subsystem is a candidate for a new upstream extension point.
