6 Subsystem Deep-Dives (original agent reports)

01 · Platform Entry + Core Scheduler
Plugin registration sequence, NPUPlatform PlatformBase impl, MoECommType taxonomy, and 4 in-house scheduler innovations — ProfilingChunk (quadratic-fit closed-form), RecomputeScheduler, ShortRequestFirst, CompressAttentionManager.
02 · Worker / Model Runner / Compilation
NPUWorker lifecycle, NPUModelRunner 5188-line hot path, v1→v2 runner split, block table for MLA/PCP/DCP, 8 torch.compile Ascend passes (QKNormRope / MatmulAllReduce / SP rewrites), ACLGraph vs torch.compile vs Triton.
03 · Attention / MLA / Ops / Triton
7 attention backends gated by (use_mla,use_sparse,use_compress). MLA via weight-absorption + fused FIA-v2. DSA/SFA/FA3/KDA/GDN. FusedMoE comm stack (MC2/AllToAll/FusedMC2). ~40 Triton kernels. 310P = forked operator tree.
04 · Distributed / KV Transfer / EPLB
HCCL communicator layer. Mooncake pull-model PD disaggregation (batch_transfer_sync_read, user-space RDMA). NPU-IPC + HCCL weight-transfer engines. EPLB 4 strategies (Random/Default/Swift/FlashLB) in a 3-phase subprocess loop.
05 · Models / Quant / Spec Decode / Loader
3-tier model matrix (only 3 custom ForCausalLM classes). DeepSeek V4 deep-dive (DSA/HC kernels). 11 speculative proposers. 20+ quant schemes. netloader (HCCL P2P) + RFork (Yuanrong RDMA seed loader). Two-tier KV-offload.
06 · Patch System / Device / Xlite
~60 monkey-patches across 6 categories (CUDA replace / Ascend-enable / hot-fix / perf-fusion / model-compat / device-fork). BaseDeviceAdaptor 1000+ line shim. CaMemAllocator sleep mode. Xlite = C++ whole-model fusion runtime. CPU binding topo modes.
How the study was done. The codebase (~128k LOC) was split into 6 subsystems along responsibility boundaries (not directory boundaries). Each subsystem was analyzed by an independent general-purpose agent writing to agent_reports/0{N}_*.md (253–530 lines each); the synthesis report and first-principles doc were produced by reading those reports + re-reading the critical code paths. Source markdown files are in ../agent_reports/.
