一、Short answer / verdict

  • The closest single academic match is HashEvict (Liu et al., arXiv:2412.16187) — it proposes exactly "binarized Gaussian projections + Hamming distance, computed pre-attention, with a lightweight binary structure in GPU memory," to identify and evict cosine-dissimilar tokens. The algorithmic core of kvcomp_attn is essentially HashEvict's selection step, promoted from a token-level eviction rule to a pre-paged-attention block retrieval + block_table rewrite.
  • The theoretical foundation is Charikar's SimHash (2002, "Similarity Estimation Techniques from Rounding Algorithms," NIPS 2002): P[sign(xR)=sign(yR)] = 1 − θ/π → Hamming distance between random-projection binary codes is a consistent estimator of angular distance.
  • H2O (arXiv:2306.14048) is NOT the origin of the hashing part. H2O is a cumulative heavy-hitter score eviction policy built from actual attention scores, with no hashing whatsoever. Hashing-based sparse work runs on a different branch (SimHash/LSH). kvcomp_attn is closer to HashEvict/SOCKET than to H2O.
  • vllm_ascend's own naming confirms the lineage: the C++ source lives in csrc/attention/kv_quant_sparse_attn_sharedkv — this is Huawei's NPU implementation of a SimHash-style quantized sparse KV attention, with the "shared-kv" / "quant" dimensions added on top of the hash-retrieval core.

三、Theoretical foundation of SimHash for attention

Charikar's SimHash identity (the whole trick in one line):
Let R ∈ ℝ^{d×b} have i.i.d. N(0,1) entries. For unit vectors x,y with angle θ,
P[ sign(xR) = sign(yR) ] = 1 − θ/π
Equivalently θ = π · (1 − P[match]) = π · (Hamming(xR,yR) / b).
So Hamming distance between b-bit random-projection codes is a consistent estimator of angular distance (variance ∝ 1/b).
  • For normalized query/key vectors, q·k = cos θ; therefore Hamming on sign codes orders nearly identically to true dot-product/cosine similarity.
  • A block's summary = one sign-code (or a small aggregated signature) is enough to rank blocks; reading full d-dim keys is unnecessary → the whole memory/compute saving.
  • Because the code is packed into uint8/uint64, Hamming becomes a fast popcount (XOR+popcnt), which maps perfectly to NPU scalar/vector units → npu_hamming_dist_top_k.
  • Error bound (the reason it generalizes): by Hoeffding, the empirical Hamming on b bits estimates θ within O(1/√b) w.h.p. The HashEvict/RaBitQCache papers each tighten this for the attention case.
Original source: Charikar, "Similarity Estimation Techniques from Rounding Algorithms," NIPS 2002. Modern attention-context restatement: see H2O/Reformer citations and the HashEvict/RaBitQCache papers above.

六、Conclusion & key takeaways

  • Origin: kvcomp_attn's Hamming-selection core descends from Charikar's SimHash (2002), filtered through the LSH-attention wave (Reformer, 2001.04451) and most immediately from HashEvict (2412.16187), with the randomized-projection-then-binary paradigm tightened by RaBitQCache (2606.31519) and the soft-collision/better-scoring angle from SOCKET (2602.06283).
  • Not H2O. H2O (2306.14048) is a different idea (accumulated real attention scores → heavy hitters). The hashing path is a separate lineage. Confusing the two is common because both produce a "small salient KV subset"; the selection mechanism is what differs.
  • Theory that matters: P[sign(xR)=sign(yR)] = 1−θ/π. All Hamming-sparse methods are just careful deployments of this identity so that Hamming distance ≈ angular distance ≈ (monotone in) attention score.
  • vs. learned NSA: hashing is parameter-free / drop-in / NPU-friendly but biased; learned indexers (NSA, the other DSA path) are data-accurate but need training. vllm_ascend implements both; kvcomp_attn is the inference-time, learn-free member of the pair.
